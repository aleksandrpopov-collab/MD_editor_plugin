# Реализация плагина MD|ed

## Архитектура (важно для «ничего не сломать»)

- **Тема** приходит из IDE: `MDEdFileEditor.kt` вызывает `initDocument({ theme: { isDark: true } })`,
  а `main.tsx` ожидает `theme.backgroundColor` / `theme.textColor`. Эти поля сейчас **не передаются** →
  применяются дефолты, а реальные цвета захардкожены в `index.css` (`#fff`, `#555`, `#3c3f41`, `#5c5c5c`,
  CodeMirror `oneDark`) и в `App.tsx` (`text-white`, `prose-invert`). Выбранный подход к теме — две
  фиксированные палитры (dark/light), переключаемые одним флагом `isDark` (см. баг №1), без воспроизведения
  произвольной цветовой схемы IDE.
- **Списки**: `StarterKit` уже включает `orderedList` (сериализация через `tiptap-markdown` работает),
  но нет пункта в slash-меню, кнопки в тулбаре и CSS под `ol`.
- **Сериализация** Markdown — через `tiptap-markdown`. Её менять не нужно; после правок стилей/расширений
  проверять round-trip dev-хелпером `window.__mdedGetMarkdown()` (см. `App.tsx`).
- **Toggle MD|ed / Markdown split** — это стандартные вкладки внизу, которые IntelliJ рисует из-за двух
  `FileEditorProvider` (наш + дефолтный). Положение «внизу» — следствие архитектуры, а не нашего кода.

## Рекомендуемый порядок выполнения (минимизация риска)

1. Баги №3, №4, №5 (CSS-отступы, выравнивание, placeholder) — локальные, низкий риск.
2. Баг №2 (нумерованные списки) — аддитивно, обязателен round-trip-тест.
3. Баг №1 (темы) — две палитры по флагу `isDark`: сперва ввести CSS-переменные и блок `[data-theme="light"]`
   (рефакторинг без смены вида), затем прокинуть `isDark` из IDE и живое переключение.
4. Улучшение (toggle наверх) — отдельно, затрагивает Kotlin-архитектуру.

# Техническое задание: Редактирование данных (Inline-редактирование в режиме просмотра)

Этот документ содержит детальное описание требований к интерактивному редактированию Markdown-документов прямо внутри окна просмотра (WYSIWYG / Notion-like редактирование).

---

## 3. Редактирование данных

Основная цель плагина — позволить пользователю изменять содержимое файла `.md`, кликая непосредственно по элементам интерфейса просмотра, без работы с исходным кодом разметки Markdown.

### 3.1. Общие принципы UX редактирования (Инлайн-редактирование)
* **Принцип "Кликнул и пиши":** 
  * При клике на любой текстовый блок (абзац, заголовок, элемент списка, цитата, ячейка таблицы) он переходит в режим редактирования. Появляется текстовый курсор (каретка).
  * Используй стандартные компоненты Intelij
  
* **Всплывающая панель форматирования (Inline Toolbar):**
  * При выделении текста мышкой появляется контекстное меню со следующими вариантами форматирования:
    * **B** (Жирный)
    * *I* (Курсив)
    * ~~S~~ (Зачеркнутый)
    * `Code` (Моноширинный код)
    * 🔗 (Вставить/изменить ссылку)
  * Клик по кнопке применяет соответствующий стиль к тексту. На уровне исходного Markdown-файла вставляются символы `**`, `*`, `~~`, `` ` `` или разметка ссылки `[text](url)`.
* **Быстрое меню добавления блоков (Notion-like `/` menu):**
  * Если каретка стоит в начале новой строки то после нее мы показываем Placeholder 'Type / for add new block'. При нажатии клавиши `/` (слэш) на новой строке открывается выпадающий список доступных блоков:
    * Заголовок 1, 2, 3
    * Tекст
    * Маркированный список
    * Чек-лист
    * Таблица
    * Блок кода
    * Цитата
    * Разделитель
    
  * Выбор пункта преобразует текущую строку в выбранный блок.
  * Если пользователь продолжает вводить текст после `/`, то список фильтруется по введенному тексту (например, `/ta` покажет только "Таблица").
  * Если пользователь введет пробел или второй символ '/' после `/`, то список закроется и будет вставлен обычный символ `/`.

* **Backspase behaviour**
  * backspace на пустой строке удаляет текущий блок и переносит фокус на предыдущий. 
  * Если это пустой пукт списка или чек-листа то backspace удаляет его и каретка переходит в предыдущий пункт списка
  * Если это единственный пукт списка то backspace удаляет спискок и переходит в предыдущий блок

### 3.2. Специфика редактирования сложных блоков

#### А. Таблицы (Tables)
* **Интерактивные кнопки управления:**
  * При наведении на таблицу сверху и слева от нее появляются панели с кнопками `+` для быстрого добавления колонок или строк.
  * Каждая ячейка таблицы редактируется независимо как текстовое поле.
  * При наведении на границы колонок курсор меняется на resizing-курсор, позволяя изменять ширину колонок перетаскиванием (drag-to-resize).
  * Кнопка удаления (иконка корзины) появляется при наведении на конкретную строку или колонку.

#### Б. Списки и Чек-листы (Lists & Checklists)
* **Интерактивные чекбоксы:**
  * При клике на чекбокс задачи `- [ ]` он меняет свое состояние на выполненный `- [x]` (и наоборот). Текст задачи при этом автоматически зачеркивается. Это действие сразу сохраняется в файл.
* **Управление структурой:**
  * Клавиша `Tab` в начале элемента списка увеличивает вложенность (добавляет отступ).
  * Клавиша `Shift + Tab` уменьшает вложенность (убирает отступ).
  * Перетаскивание (Drag-and-Drop) элементов списка мышкой для изменения их порядка (опционально, но рекомендуется для премиум-UX).
  * Нажатие `Enter` в конце элемента списка создает новый элемент списка того же типа на следующей строке.

#### В. Блоки кода (Code Blocks)
* **Встроенный редактор кода:**
  * Блок кода рендерится с использованием **CodeMirror 6** прямо внутри веб-интерфейса TipTap. Это обеспечивает полноценную подсветку синтаксиса, автодополнение базовых конструкций и нативную работу в браузере без наложения тяжеловесных Java-компонентов.
  * В правом верхнем углу блока отображается иконка копирования, при нажатии на которую код копируется в буфер обмена, пользователь видит всплывающее уведомление 'code copied to clipboard'.
#### Г. Ссылки и Изображения (Links & Images)
* **Редактирование ссылок:**
  * При клике на ссылку открывается мини-попап с полями: "Текст ссылки" и "Адрес URL", а также кнопкой "Перейти" и "Удалить ссылку".
* **Редактирование изображений:**
  * Поддержка перетаскивания (Drag & Drop) графических файлов напрямую в область просмотра. Перехват события осуществляется на стороне Java (через `DropTargetListener` поверх JCEF). Java-код копирует изображение в папку проекта (`images/`) и отправляет в JS команду на вставку разметки `![alt](images/filename.png)`. Это позволяет избежать передачи больших файлов через JS-мост.
  * При клике на изображение появляются маркеры по углам для изменения размера (resizing).

### 3.3. Синхронизация данных и сохранение

* **Двусторонняя автосинхронизация (Bi-directional Sync):**
  * Изменения, внесенные в интерактивном режиме просмотра, должны сохраняться в исходный `.md` файл.
  * Чтобы избежать зависаний интерфейса, запись в файл должна происходить в фоновом потоке с использованием механизма Debounce (задержка около 300–500 мс после прекращения ввода).
  * Если файл параллельно изменяется во внешнем редакторе (или стандартном текстовом редакторе IntelliJ), плагин должен автоматически обновить визуальное представление без перезагрузки вкладки и без потери текущего фокуса ввода (если это возможно).
* **Интеграция с системой Undo/Redo:**
  * Мелкогранулярная история действий (ввод символов, применение стилей) управляется **внутренним модулем истории TipTap** в веб-слое. При нажатии `Ctrl+Z` (`Cmd+Z`) внутри браузера срабатывает отмена на уровне TipTap.
  * Синхронизация с историей команд IDE (`CommandProcessor` в IntelliJ) происходит при дебаунс-сохранении. Каждое такое сохранение создает в IDE одну укрупненную точку отката.

### 3.4. Горячие клавиши (Keyboard Shortcuts)

Для быстрого редактирования должны поддерживаться следующие клавиатурные сочетания:

| Сочетание клавиш (Win / Mac) | Действие |
| :--- | :--- |
| `Ctrl + B` / `Cmd + B` | Сделать выделенный текст жирным |
| `Ctrl + I` / `Cmd + I` | Сделать выделенный текст курсивом |
| `Ctrl + K` / `Cmd + K` | Превратить выделенный текст в ссылку |
| `Ctrl + Shift + C` / `Cmd + Shift + C` | Превратить строку в блок кода |
| `Enter` | Создать новый абзац или элемент списка |
| `Shift + Enter` | Перенос строки внутри текущего блока (без создания нового абзаца) |
| `Tab` / `Shift + Tab` | Изменение уровня вложенности списка |

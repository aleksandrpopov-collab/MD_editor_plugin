# Техническое задание: Интерактивный Markdown-редактор для IntelliJ IDEA

Данный документ описывает требования к плагину для сред разработки на базе IntelliJ Platform (IntelliJ IDEA, WebStorm, PyCharm и др.), который позволяет просматривать файлы формата `.md` (Markdown) в красивом отрендеренном виде и редактировать их содержимое непосредственно в режиме просмотра (WYSIWYG / Live Editing), без необходимости переключения между вкладками "Editor" и "Preview".

---

## 1. Открытие плагина

Этот блок описывает сценарий запуска и интеграции плагина в интерфейс IntelliJ I
DEA.

### 1.1. Интеграция в контекстное меню
* **Действие пользователя:** Клик правой кнопкой мыши (ПКМ) по любому `.md` файлу в панели проекта (Project Tool Window) или по открытой вкладке редактора `.md` файла.
* **Пункт меню:** В контекстное меню должен добавляться новый пункт: **"Open at MD|ed"**.

### 1.2. Открытие редактора
* **Поведение при открытии:** 
  * При выборе пункта меню файл открывается в отдельной вкладке **FileEditor** 
  * Имя вкладки совпадает с именем файла (например, `README.md`).
  * Если файл уже открыт, повторный клик ПКМ и выбор пункта меню должен просто переносить фокус на уже существующую вкладку с этим файлом, а не открывать новую копию.
* **Управление вкладками:** Вкладки можно закрывать кликом на крестик.
* **Сохранение состояния:** При перезапуске IDE открытые вкладки плагина и их расположение должны восстанавливаться (если файлы не были удалены).

## 2. Отображение данных (Рендеринг блоков Markdown)

| Тип блока / Элемент | Пример синтаксиса Markdown | Описание визуального отображения (Заполняется вами)                                                                                                                                                                                                                                                                                                                                                                                       | Дополнительные требования |
| :--- | :--- |:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------| :--- |
| **Заголовки (H1 - H6)** | `# Заголовок 1`<br>`## Заголовок 2` | Рендерятся в теги `<h1>`–`<h6>`. Размер зависит от уровня: H1 — `font-size: 1.5rem`, H2 — `font-size: 1rem`, H3–H6 — `font-size: 0.8rem`. Все уровни: `font-weight: 600`, `line-height: 1.3`, `color: var(--heading-color)`. Общие отступы: `margin: 1rem 0 0.75rem`. Дополнительные подчёркивания и `border-bottom` не применяются.                                                                                                                                                                   | Поддержка автоматического создания оглавления (TOC). |
| **Обычный текст (Абзацы)** | `Это обычный текст абзаца.` | Рендерится в тег `<p>`. Базовый текст редактора наследует настройки контейнера `.ProseMirror`: `color: var(--text-color)`, `font-size: 13px`, `line-height: 1.5`. Для абзацев применяется `font-size: 0.8rem`, `line-height: 1.5`, `margin-bottom: 0.5rem`. Шрифт наследуется от родительского контейнера (`sans-serif`). Пустая строка между блоками текста создаёт новый `<p>`.                                                                                                                                                | Поддержка мягкого переноса строк. |
| **Жирный / Курсив / Зачеркнутый** | `**жирный**`, `*курсив*`, `~~зачеркнутый~~` | `**text**` → `<strong>`, `font-weight: bold`. `*text*` → `<em>`, `font-style: italic`. `~~text~~` → `<del>`, `text-decoration: line-through`. Элементы могут быть вложены друг в друга без ограничений.                                                                                                                                                                                                                                   | Возможность комбинирования стилей. |
| **Маркированный список** | `- Элемент 1`<br>`- Элемент 2` | Рендерится в `<ul><li>`. `list-style-type`: уровень 1 — `disc`, уровень 2 — `circle`, уровень 3+ — `square`. `padding-left: 2em` на каждый уровень вложенности. `margin-bottom: 0.25em` между элементами.                                                                                                                                                                                                                                 | Красивые кастомные маркеры (точки, квадраты). |
| **Нумерованный список** | `1. Первый`<br>`2. Второй` | Рендерится в `<ol><li>`. `list-style-type: decimal`. `padding-left: 2em`. Нумерация сквозная внутри одного `<ol>`; при вложенности нумерация дочернего `<ol>` начинается с 1. Атрибут `start` поддерживается для задания начального номера.                                                                                                                                                                                               | Автоматическое выравнивание номеров по правому краю. |
| **Списки задач (Checklists)** | `- [ ] Задача`<br>`- [x] Выполнено` | Рендерится в `<ul>` со встроенным `<input type="checkbox">` перед текстом элемента. `[ ]` → `checked=false`, `[x]` → `checked=true`. Выполненный элемент: `color: <muted-color>`, `text-decoration: line-through`.                                                                                                                                                                                                                        | Интерактивные чекбоксы, на которые можно кликать для изменения статуса. |
| **Блоки кода** | ` ```javascript`<br>`const a = 5;`<br>```` ` | Рендерится в `<pre><code class="language-*">`. `font-family: monospace`, `font-size: 0.9em`. Фон: `background: <code-bg-color>`, `border-radius: 4px`, `padding: 1em`. Подсветка синтаксиса применяется через токенизатор (Prism.js или аналог) по значению атрибута `class`. Метка языка — абсолютно позиционированный `<span>` в правом верхнем углу. Кнопка копирования — `<button>` в правом верхнем углу, `z-index` выше блока кода. | Подсветка синтаксиса (Syntax Highlighting) в зависимости от языка, кнопка "Копировать" в углу блока. |
| **Встроенный код** | `` `const a = 5` `` | Рендерится в `<code>`. `font-family: monospace`, `font-size: 0.875em`. `background: <inline-code-bg>`, `border-radius: 3px`, `padding: 0.2em 0.4em`. Цвет текста может задаваться отдельным CSS-токеном темы. Не переносится на новую строку (`white-space: nowrap`).                                                                                                                                                                     | Выделение фоном (светло-серый или темный) и моноширинным шрифтом. |
| **Цитаты (Blockquotes)** | `> Важная цитата` | Рендерится в `<blockquote>`. `border-left: 4px solid <accent-color>`, `padding-left: 1em`, `margin-left: 0`. `color: <muted-text-color>`. Поддерживается вложенность (`>> ...`) — каждый уровень добавляет вложенный `<blockquote>` с тем же стилем.                                                                                                                                                                                      | Вертикальная цветная линия слева (акцентный цвет), курсивное начертание текста. |
| **Горизонтальный разделитель** | `---` | Рендерится в `<hr>`. `border: none`, `border-top: 1px solid <border-color>`, `margin: 1.5em 0`. Занимает 100% ширины родительского контейнера.                                                                                                                                                                                                                                                                                            | Тонкая линия серого цвета с отступами сверху и снизу. |
| **Ссылки** | `[Google](https://google.com)` | Рендерится в `<a href="...">`. `color: <link-color>`, `text-decoration: none`. При `hover`: `text-decoration: underline`, `cursor: pointer`. `title`-атрибут содержит полный URL и отображается как нативный браузерный tooltip. Внешние ссылки открываются с `target="_blank" rel="noopener noreferrer"`.                                                                                                                                | Цветные ссылки, подчеркивание при наведении. Подсказка (tooltip) с полным URL. |
| **Изображения** | `![Описание](path/image.png)` | Рендерится в `<img src="..." alt="...">`. `max-width: 100%`, `height: auto`, `display: block`. Alt-текст отображается как `<figcaption>` под изображением или как значение `alt` при недоступности ресурса. Поддерживаются относительные пути (резолвятся относительно файла `.md`) и абсолютные URL.                                                                                                                                     | Отображение картинки с возможностью масштабирования по ширине окна. |
| **Таблицы** | `| Колонки                                                                                                                                                                                                                                                                                                                                                                                                                                   |`<br>`|---|` | Рендерится в `<table>`. `border-collapse: collapse`. Ячейки: `border: 1px solid <border-color>`, `padding: 0.5em 1em`. Строка заголовка (`<thead> <th>`): `font-weight: bold`, `background: <header-bg-color>`. Чётные строки `<tbody> <tr>:nth-child(even)`: `background: <zebra-bg-color>`. Выравнивание ячейки задаётся через `text-align` согласно синтаксису выравнивания (`:---` → left, `:---:` → center, `---:` → right). | Границы ячеек, выделение заголовков, чередование цвета строк (zebra striping). |
| **Математические формулы** | `$$E = mc^2$$` | Блочные формулы (`$$...$$`) рендерятся в `<div class="math-block">`, `display: block`, `text-align: center`, `margin: 1em 0`. Строчные (`$...$`) — в `<span class="math-inline">`, встраиваются в поток текста. Рендеринг выполняется библиотекой KaTeX (предпочтительно) или MathJax. При ошибке парсинга отображается исходный текст формулы с визуальным индикатором ошибки.                                                           | Корректный рендеринг формул (например, с помощью KaTeX или MathJax). |
| **Сноски** | `Текст[^1]` | Маркер в тексте рендерится в `<sup><a href="#fn-1" id="fnref-1">[1]</a></sup>`. `font-size: 0.75em`, `color: <link-color>`. Блок сносок размещается в конце документа в `<section class="footnotes">`, отделён `<hr>`. Каждая сноска — `<li id="fn-1">` с обратной ссылкой `<a href="#fnref-1">↩</a>`. Клик по маркеру — прокрутка к сноске через якорную навигацию (`scrollIntoView`).                                                   | Кликабельные надстрочные индексы с переходом к тексту сноски внизу страницы. |
